# Deep Linking Setup (Custom URL Scheme)

## Что это даёт?

Когда пользователь делится челленджем, ссылка `beraw://challenge/123` будет:
- Открывать приложение напрямую (если установлено)
- Открывать конкретный челлендж в приложении

## Формат ссылок:

```
beraw://challenge/UUID-челленджа
```

Пример:
```
beraw://challenge/A1B2C3D4-E5F6-7890-ABCD-EF1234567890
```

## Код уже добавлен:

✅ `DynamicLinksManager.swift` - создание и обработка ссылок  
✅ `AppDelegate.swift` - интеграция с app lifecycle  
✅ `ShareChallengeSheet` - генерация ссылок  
✅ `RootView` - навигация к челленджу  
✅ `Info.plist` - Custom URL Scheme: `beraw`

## Как это работает:

1. Пользователь делится челленджем → создаётся ссылка `beraw://challenge/123`
2. Получатель (с установленным приложением) кликает на ссылку
3. iOS открывает приложение и передаёт URL
4. `AppDelegate` перехватывает URL и передаёт в `DynamicLinksManager`
5. Приложение автоматически открывает нужный челлендж

## Тестирование:

### На устройстве с установленным приложением:

1. Открой челлендж в приложении
2. Нажми "Share"
3. Скопируй ссылку (формат: `beraw://challenge/...`)
4. Отправь себе в Notes/Messages
5. Нажми на ссылку → приложение должно открыться и показать челлендж

### Через Safari:

1. Открой Safari
2. Введи в адресную строку: `beraw://challenge/твой-UUID`
3. Нажми Enter → Safari предложит открыть приложение

## Ограничения:

⚠️ **Требуется установленное приложение**  
Если у получателя нет приложения, ссылка не сработает (iOS покажет ошибку)

⚠️ **Не работает через веб**  
Custom URL Scheme не работают как обычные HTTPS ссылки

⚠️ **Не работает в некоторых мессенджерах**  
Некоторые мессенджеры блокируют custom schemes
